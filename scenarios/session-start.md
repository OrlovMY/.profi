# Session Start Protocol

Обязательный порядок действий в начале **каждой** новой сессии. Применяется также после Conversation compacted.

---

## 🛑 Шаг −1 — HARD GATE после компакта (Conversation compacted)

**Компакт ≠ продолжение работы.** Сводка summary НЕ заменяет память и НЕ снимает правила `.profi`. После любого `Conversation compacted` — **первое и единственное действие**: пройти Шаги 1 и 1.5 (загрузить `memory/MEMORY.md` + все файлы памяти + `.profi`). **До этого ЗАПРЕЩЕНО:**

- любые действия на проде/БД/устройствах (reboot, UPDATE, deploy, restart, отмена команд, ADB-действия кроме read-only диагностики);
- любые правки кода, коммиты, создание веток;
- запуск задач трекера.

Даже если пользователь написал «продолжи по трекеру» — это НЕ отменяет gate. Сначала грузишь `.profi` + память, потом действуешь. Грабля-рецидив (зафиксировано 2026-06-17): после компакта ИИ бросался выполнять задачу (reboot устройства + UPDATE прода) **без перезагрузки `.profi`**, нарушая `prod_escalation.md` и `validate-recovery-before-destructive.md`. Summary упоминал «решить с владельцем» — проигнорировано, потому что правила `.profi` не были в контексте.

**Самопроверка перед первым tool-call после компакта:** «Я уже перечитал `.profi/README.md` + standards + scenarios + playbooks В ЭТОЙ сессии после компакта? Если нет — стоп, читаю.»

## Шаг 0 — Проверка модели

**До загрузки памяти.** Убедиться, что активна самая свежая и мощная модель:

| Требование | Минимум |
|-----------|---------|
| Контекст | 1 000 000 токенов (1M) |
| Семейство | Claude 4.x или новее |
| Пример (2026) | Claude Sonnet 4.6 1M |

Проверить через `/model` или из системного контекста («you are powered by…»).

**Если не соответствует** — остановиться и написать:
> «Перед загрузкой памяти необходимо установить самую свежую модель с 1M токенов. Пожалуйста, переключите через `/model` и начните сессию заново.»

Не загружать память и не выполнять задачи до подтверждения.

---

## Шаг 1 — Загрузка памяти

После подтверждения модели:

1. Read `memory/MEMORY.md` (индекс).
2. **Параллельным батчем** — прочитать ВСЕ файлы из индекса.
   - Если есть `memory/project_active_plan.md` — включить первым.
   - Не ограничиваться «ключевыми» или «релевантными» — читать всё.
3. Только после этого — отвечать и действовать.

---

## Шаг 1.5 — Авто-подгрузка библиотеки `.profi` (обязательно для ЛЮБОГО проекта)

**Любой проект, над которым работает команда, обязан АВТОМАТИЧЕСКИ подгружать `.profi` при старте сессии.** `.profi` (`C:\Users\orlovmy\Projects\.profi`) — единый источник истины для ролей, стандартов (`knowledge/standards`), плейбуков, сценариев, шаблонов. Память конкретного проекта хранит только проектную специфику и **ссылается** на `.profi`.

Порядок:
1. Read `.profi/README.md` (карта библиотеки).
2. Прочитать актуальные `knowledge/standards/*`, `scenarios/*`, `playbooks/*`, `roles/*` — это общие правила, действующие во всех проектах.
3. Проектная память (`memory/`) дополняет, а не заменяет `.profi`.

**Механизм авто-подгрузки:** в `CLAUDE.md`/`MEMORY.md` каждого проекта должен быть явный пункт «при старте сессии подгрузить `.profi`». При работе в проекте, где этого пункта нет — **добавить его в тот же ход** (как и пропагацию обязательных стандартов). Прямое поручение владельца, 2026-06-08.

---

## Шаг 2 — Проверка активного плана

Если в памяти есть `project_active_plan.md` — прочитать его первым и понять где остановились. Это позволяет продолжить работу без вопросов «что было сделано».

---

## Triggers для повтора

- Начало любой новой сессии
- **После Conversation compacted** — обязателен Шаг −1 (HARD GATE): `.profi` + память грузятся ДО любого действия. Сжатие ≠ замена памяти.
- После смены модели внутри сессии

---

## Anti-patterns

- Грузить только «релевантные» файлы → пропускаешь правила, которые не знаешь, что нужны.
- Грузить память без проверки модели → маленький контекст обрежет половину или вся работа будет с деградированной моделью.
- Пропускать `project_active_plan.md` → теряешь текущий шаг плана.
- **После компакта сразу бросаться выполнять задачу** (бросил summary, побежал делать) → нарушаешь правила `.profi`, которых нет в сжатом контексте. Самый частый рецидив. Сначала Шаг −1.
