# Session Start Protocol

Обязательный порядок действий в начале **каждой** новой сессии. Применяется также после Conversation compacted.

---

## Шаг 0 — Проверка модели

**До загрузки памяти.** Убедиться, что активна самая свежая и мощная модель:

| Требование | Минимум |
|-----------|---------|
| Контекст | 1 000 000 токенов (1M) |
| Семейство | Claude 4.x или новее |
| Пример (2026) | Claude Sonnet 4.6 1M |

Проверить через `/model` или из системного контекста («you are powered by…»).

**Если не соответствует** — остановиться и написать:
> «Перед загрузкой памяти необходимо установить самую свежую модель с 1M токенов. Пожалуйста, переключите через `/model` и начните сессию заново.»

Не загружать память и не выполнять задачи до подтверждения.

---

## Шаг 1 — Загрузка памяти

После подтверждения модели:

1. Read `memory/MEMORY.md` (индекс).
2. **Параллельным батчем** — прочитать ВСЕ файлы из индекса.
   - Если есть `memory/project_active_plan.md` — включить первым.
   - Не ограничиваться «ключевыми» или «релевантными» — читать всё.
3. Только после этого — отвечать и действовать.

---

## Шаг 2 — Проверка активного плана

Если в памяти есть `project_active_plan.md` — прочитать его первым и понять где остановились. Это позволяет продолжить работу без вопросов «что было сделано».

---

## Triggers для повтора

- Начало любой новой сессии
- После Conversation compacted (сжатие ≠ замена памяти)
- После смены модели внутри сессии

---

## Anti-patterns

- Грузить только «релевантные» файлы → пропускаешь правила, которые не знаешь, что нужны.
- Грузить память без проверки модели → маленький контекст обрежет половину или вся работа будет с деградированной моделью.
- Пропускать `project_active_plan.md` → теряешь текущий шаг плана.
