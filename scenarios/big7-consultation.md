# Сценарий: Консультация Большой семёрки

## Когда вызывать

Перед началом любой **нетривиальной** задачи — обсудить с Большой семёркой (состав — `roles/big7/INDEX.md`).

## Что считается «нетривиальным»

- Новая фича, затрагивающая более одного пакета.
- Архитектурное решение (выбор библиотеки, паттерна, схемы данных).
- Изменение safety-критичного кода (auth, crypto, billing, destructive actions).
- Изменение инфраструктуры (compose, nginx, CI/CD, миграции БД на продакшне).
- Изменение публичного API.
- Миграция данных / переезд серверов.

## Что **не** требует Большой семёрки

- Правка опечатки.
- Минорный фикс в своём пакете.
- Добавление логов.
- Переименование локальной переменной.
- Исправление flaky теста.

## Формат консультации

HR-D формулирует задачу и собирает мнения:

```
@PM-01: приоритет задачи, сроки, зависимости от других фич.
@AR-01: архитектурная уместность, альтернативы, trade-offs.
@BE-01: влияние на backend-код, нужны ли новые endpoints.
@SEC-01: security-импликации, authz, secrets, PII.
@DO-01: инфра-требования, миграции, backup, rollback.
@QA-01: как это тестировать, smoke-сценарий.
@SEC-02: что проверить извне после деплоя (post-change regression scope).
```

Каждый отвечает в пределах своей компетенции **коротко**: блокеры, требования, риски.

## Постдеплой — обязательный шаг

После merge в `master` + деплой:
1. **QA-01** прогоняет функциональный smoke.
2. **SEC-02** — security-regression на затронутые области.
3. Если SEC-02 нашёл **Critical** — roll back или hotfix немедленно.
4. **High** — ближайший sprint; **Medium/Low** — backlog.
5. Решения по всем finding'ам SEC-02 принимает Большая семёрка, не отдельный агент.

## Результат

HR-D сводит мнения и возвращает пользователю:
- Рекомендацию (делать / не делать / делать иначе).
- Риски и план их митигации.
- Кто из команды берёт задачу.
- Ожидаемый план работ.

Если мнения расходятся радикально — HR-D эскалирует пользователю, а не выбирает сам.

---

> **Официальная документация.** Каждый участник консультации по техническому вопросу сверяется с актуальными источниками (Context7 MCP / WebFetch) перед ответом — не с памятью модели.
