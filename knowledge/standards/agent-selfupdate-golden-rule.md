# 🥇 ЗОЛОТОЕ ПРАВИЛО: агент обязан ставиться И самообновляться (все ОС)

**Класс:** HARD / GOLDEN. **Действует для:** агентов под Windows, Android, macOS, Linux и embedded/IoT-устройства.

## Правило
Перед релизом (раскатка на флот) ЛЮБОЙ новой версии агента ОБЯЗАТЕЛЬНО доказать на стенде ТРИ вещи:
1. **Чистая установка** версии N проходит на целевой ОС.
2. **Forward self-update:** версия N умеет САМА обновиться на N+1 (механизм автообновления в N рабочий).
3. **Механизм не сломан вперёд:** обновление N→N+1 стартует и завершается БЕЗ ручного вмешательства (без переустановки руками, без физического доступа).

**Релиз без прохождения всех трёх = заблокирован.** Это не «протестировали инсталл» — обязательно проверяется именно способность обновиться ИЗ этой версии в будущую.

## Почему
Типичный отказ: новая версия агента **перестаёт самообновляться** → требуется ещё одна версия, чтобы починить автообновление. Но устройство может быть **недоступно** (мобильные/embedded-устройства уходят offline, удалённые машины, нет физ.доступа) — и тогда ни обновить, ни переустановить агента уже нельзя → **потеря управления устройством навсегда**. Сломанный self-update = тихий кирпич управляемости.

## Почему ломается (известные корни — проверять адресно)
- **Windows:** Session-0 isolation службы; Authenticode/подпись блокирует подмену бинаря при self-update; автозапуск через реестр vs служба; рассинхрон схем версионирования (version_code).
- **Android:** AccessibilityService/Device-Admin **unbind после APK replace**; silent install через PackageInstaller (Device Owner); targetSdk/permission gates; рассинхрон схем версионирования.
- **Embedded / OTA:** номер OTA-сборки строго растёт; залипший слот авто-апдейта; root-супервизор, перезапускающий процесс.
- **Все:** модульный self-update — КАЖДЫЙ модуль (основной агент / watchdog / tray-компонент / …) обновляется независимо.

## Обязательная тест-матрица (на КАЖДУЮ ОС, перед релизом)
| Шаг | Проверка | Доказательство |
|-----|----------|----------------|
| T1 | Чистый install N | сервис/процесс жив, чек-ин в панели управления |
| T2 | N → N+1 self-update (команда/каталог/авто) | в панели версия сменилась на N+1 БЕЗ ручных действий |
| T3 | N+1 → N+2 (цепочка не деградирует) | повтор T2 со следующей версии |
| T4 | self-update при «недоступности» канала | агент дожимает обновление после восстановления связи, не теряет апдейт |
| T5 | модульность | обновление одного модуля не ломает остальные |

Тестовые стенды: по одному реальному устройству на каждую целевую ОС/платформу. Где возможно — e2e self-update тест в CI.

## Владение / гейт
Регламент утверждается архитектурным/ревью-гейтом команды; владелец тестов self-update — QA совместно с лидами по соответствующим платформам (core/мобайл и desktop/embedded). Чек-лист self-update прикладывается к каждому релизному заданию агента.
