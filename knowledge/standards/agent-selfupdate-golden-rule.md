# 🥇 ЗОЛОТОЕ ПРАВИЛО: агент обязан ставиться И самообновляться (все ОС)

**Класс:** HARD / GOLDEN. **Появился:** owner-директива 2026-06-04 (MDMy). **Действует для:** Windows, Android, macOS, Linux, CPE-агентов.

## Правило
Перед релизом (status=finish, раскатка на флот) ЛЮБОЙ новой версии агента ОБЯЗАТЕЛЬНО доказать на стенде ТРИ вещи:
1. **Чистая установка** версии N проходит на целевой ОС.
2. **Forward self-update:** версия N умеет САМА обновиться на N+1 (механизм автообновления в N рабочий).
3. **Механизм не сломан вперёд:** обновление N→N+1 стартует и завершается БЕЗ ручного вмешательства (без переустановки руками, без физического доступа).

**Релиз без прохождения всех трёх = заблокирован.** Это не «протестировали инсталл» — обязательно проверяется именно способность обновиться ИЗ этой версии в будущую.

## Почему (обоснование owner)
Неоднократно новая версия Windows/Android-агента **переставала самообновляться** → требовалась ещё одна версия, чтобы починить автообновление. Но устройство может быть **недоступно** (CPE-модемы уходят offline, удалённые машины, нет физ.доступа) — и тогда ни обновить, ни переустановить агента уже нельзя → **потеря управления устройством навсегда**. Сломанный self-update = тихий кирпич управляемости.

## Почему ломается (известные корни — проверять адресно)
- **Windows:** Session-0 isolation службы; Authenticode/подпись блокирует подмену бинаря при self-update; HKLM\Run vs служба; version_code в 2 схемах. (см. `feedback_windows_agent_deploy_playbook`)
- **Android:** AccessibilityService/Device-Admin **unbind после APK replace**; DO PackageInstaller silent install; targetSdk/permission gates; version_code 2 схемы. (см. `project_accessibility_unbind`, `feedback_modular_autoupdate`)
- **CPE:** OTA `.upd` build строго растёт; залипший слот авто-апдейта; root_daemon supervisor. (см. `project_cpe_naming_and_slot`)
- **Все:** модульный self-update — КАЖДЫЙ модуль (Agent/Guard/Tray/…) обновляется независимо.

## Обязательная тест-матрица (на КАЖДУЮ ОС, перед finish)
| Шаг | Проверка | Доказательство |
|-----|----------|----------------|
| T1 | Чистый install N | сервис/процесс жив, чек-ин в админке |
| T2 | N → N+1 self-update (команда/каталог/авто) | в админке версия сменилась на N+1 БЕЗ ручных действий |
| T3 | N+1 → N+2 (цепочка не деградирует) | повтор T2 со следующей версии |
| T4 | self-update при «недоступности» канала | агент дожимает обновление после восстановления связи, не теряет апдейт |
| T5 | модульность | обновление одного модуля не ломает остальные |

Стенды: Windows `TLab-Test-01`; Android Meizu/тест; CPE UFI-тест. Где возможно — e2e self-update тест в CI.
