# Standard: никаких тихих установок софта

**Статус:** обязательный · **Применимость:** все проекты/среды.

## Правило
**НЕ устанавливать системный софт без явного «да» пользователя** — даже в режиме «не задавай вопросов»/автономном. Сюда входит: `winget`/`choco`/`apt`/`brew`/`pip install` глобально/`npm -g`/MSI/любой системный пакет-менеджер, драйверы, системные службы.

- Установка меняет состояние машины пользователя необратимо/широко → требует явного согласия.
- Режим «делай без уточнений» покрывает действия В рамках проекта, НЕ установку системного ПО на хост.
- Можно без спроса: локальные зависимости проекта в изолированную папку/venv проекта (не глобально), если это часть задачи.

## Зачем
Тихая системная установка = неожиданное изменение окружения пользователя (конфликты версий, мусор, безопасность). «Не задавай вопросов» ≠ «делай необратимое с хостом».

## Связано
- `no-confirm-reasonable-nondestructive` (граница: установка ≠ разумный неразрушающий шаг) · `validate-recovery-before-destructive`.

## Источник
Глобальное правило пользователя: не ставить winget/choco/MSI/pip-global и т.п. без явного «да», даже в no-questions режиме.
