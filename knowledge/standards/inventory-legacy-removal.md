# Standard: удаление legacy-совместимости — ТОЛЬКО data-driven (HARD)

**Статус:** ОБЯЗАТЕЛЬНЫЙ · **Введён:** 2026-06-01 (owner, жёстко) · **Область:** tolerant-reader / совместимость форматов во всех проектах

Это правило существует, чтобы **после старта новой сессии не натворить дичь** с удалением совместимости. Читать ПЕРЕД любым действием над legacy/compat-слоями.

## ❌ ОТ ЧЕГО ОТКАЗАТЬСЯ (НЕ делать)
1. **НЕ вырывать tolerant-reader / compat-слой «ради чистоты».** Он стоит в рантайме ~ноль (пара map-lookup) и это правильная практика (закон Постела). «Меньше строк» — не причина трогать прод.
2. **НЕ переписывать ядро (горячий checkin-путь, колоночные контракты) ради переименования полей.** Это риск регресса по ВСЕМУ парку при нулевой выгоде по данным. Конкретно запрещено: INV-7-подобная правка checkin-body/колонок ради serial_number→serial и т.п.
3. **НЕ удалять алиас «на глаз»/по предположению, что им никто не пользуется.** Только по факту (см. ниже).
4. **НЕ пересобирать агентов всего парка ради косметики имён**, если данные уже корректны (их сворачивает сервер).

## ✅ КАК ПРАВИЛЬНО (единственный допустимый путь)
1. **Инструментировать срабатывания.** Каждый раз, когда сервер сворачивает legacy→канон, писать в `inventory_alias_hits (canonical, legacy_key, hit_count, last_hit_at)` (UPSERT). Уже реализовано в `normalizeInventoryAliases` + `mirrorCellularAttrs` (миграция 146).
2. **Решение об удалении — на данных.** Алиас можно удалять, ТОЛЬКО если:
   - нет строки в `inventory_alias_hits` ИЛИ `last_hit_at < now() - 7 days`, И
   - прошло ≥1 недели с момента деплоя инструментирования (накоплены данные).
3. **Удалять по одному**, не пачкой; только из attributes-нормализатора (`renameAliases`/`unitDerivations`), **никогда** из checkin-body/колоночного контракта.
4. После удаления — проверить, что соответствующие устройства не потеряли поле (одно устройство-образец каждой платформы).

## Запрос для проверки (раз в неделю)
```sql
-- какие алиасы ещё живы (били за последние 7 дней)
SELECT canonical, legacy_key, hit_count, last_hit_at
FROM inventory_alias_hits WHERE last_hit_at > now() - interval '7 days'
ORDER BY last_hit_at DESC;
-- кандидаты на удаление = алиасы из renameAliases/unitDerivations, которых
-- НЕТ в этом списке (никто не слал ≥7 дней).
```
