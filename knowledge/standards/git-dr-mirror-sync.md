# Standard: DR-зеркало git — синхронно в оба remote

**Статус:** обязательный · **Применимость:** любой проект с зеркалом репозитория (disaster recovery).

## Правило
Если у репозитория есть DR-зеркало (второй remote: GitHub↔GitLab и т.п.) — после **каждого** push в основной remote немедленно пушить и в зеркало. «Потом» = окно рассинхрона = потеря при сбое основного хоста.

```
git push origin <branch> && git push <mirror> <branch>
```

- Конкретные remote-URL и имена зеркал — в памяти/доке проекта, НЕ в этом универсальном правиле.
- Держать одинаковый набор веток (или явно знать, что зеркалится только основная).

## Зачем
DR-зеркало имеет смысл только синхронным. Рассинхронизированное зеркало создаёт ложное чувство защищённости.
