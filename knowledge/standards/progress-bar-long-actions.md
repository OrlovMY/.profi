# Standard: прогрессбар + ETA для долгих действий

**Статус:** обязательный · **Применимость:** все проекты; любые длительные операции.

## Правило
Для **любого долгого действия** (массовая заливка/скачивание, backfill, сборка, миграция большого объёма, длинный цикл по элементам, ожидание внешнего процесса) показывать **текстовый прогрессбар + оценку времени до завершения (ETA)**.

## Формат
```
[██████░░░░░░░░] 35/46 (76%) · ~0.11 МБ/с · осталось 115МБ · ETA ~18 мин
```
- доля выполнено (N/total и %) + ASCII-бар;
- текущая скорость (если измерима);
- **ETA = остаток / скорость**.

## Как считать
- **По байтам/объёму, а не по числу шагов**, если элементы разного размера (крупные искажают «N из M»). Пример: backfill из мелких + 13МБ-файлов — считать по сумме байт залито/осталось.
- Скорость — из реального замера (не «на глаз»). Для сети — отдельным замером upload/download (они могут отличаться в разы: напр. Yandex WebDAV ~16МБ/с чтение vs ~0.11МБ/с запись).
- Остаток × скорость → ETA в минутах.

## Когда обновлять
- При запуске долгой операции прикинуть объём → стартовый бар + ETA.
- Обновлять на запрос статуса («как дела») и/или на значимых шагах. **Не молотить токенами** частым поллингом — разумный интервал.
- Для фонового ожидания — фоновый монитор, который уведомит о завершении (а не активный поллинг).
