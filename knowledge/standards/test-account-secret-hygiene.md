# Standard: гигиена секретов тест-учёток

**Статус:** обязательный · **Применимость:** любой проект с тест-окружением/тест-УЗ.

## Правило
Сброс или создание тест-учётки (пароль, токен) — **в тот же ход** записать новое значение в единый доку-артефакт проекта (напр. `docs/.../test-accounts.md`) и закоммитить. НЕ «потом», НЕ держать в голове.

- При сбросе — сбрасывать на значение **из доки** (детерминированно), а не на случайное «в уме».
- НЕ подбирать забытый пароль перебором (часто есть rate-limit/лок/классификатор).
- Доку с тест-кредами держать в репозитории ПРОЕКТА (приватном), НЕ в общей кросс-проектной библиотеке.

## ⛔ Граница (важно)
Это правило — про **процесс** (фиксировать сразу). Сами значения паролей/токенов живут ТОЛЬКО в приватной памяти/доке проекта, **никогда** в общей `.profi` (см. критерий: «можно ли опубликовать в open-source без последствий? нет → только проект»).

## Зачем
Незаписанный сброшенный пароль = потерянный доступ к тест-УЗ + время на повторный сброс/расследование.

## Источник
Реальный проект: HARD-правило после повторных потерь доступа к тест-УЗ (пароль сброшен, не записан).
