# Guide: убедиться, что деплой РЕАЛЬНО доехал (а не «запушил = задеплоил»)

## Условия применения
После деплоя фронта/сервиса нужно подтвердить, что именно ТВОЁ изменение в проде — до того как звать задачу закрытой. Особенно когда визуальная проверка дорога (нужен логин/спец-роль/спец-устройство).

## Суть (дёшево → дорого)
1. **Артефакт-grep (без логина, конклюзивно для статики/SPA):** скачать реально отдаваемый бандл и найти в нём свою строку/ключ.
   `curl -sk https://app/ | grep -oE 'index-[A-Za-z0-9_-]+\.js'` → взять имя → `curl -sk https://app/assets/<bundle> | grep -c 'МойНовыйКлюч'`.
   Если строка есть — код в проде. Если нет — деплой не доехал (старый бандл / сборка из старого src).
2. **Bundle-hash сменился?** Если имя бандла = прежнему → пересборка не произошла (частая причина: исходники на сервере не обновились перед `build`).
3. **Версия/health:** `curl /healthz` 200; для API — эндпоинт версии/нового роута (404→старый бинарь).
4. **Только потом — визуал** (логин, открыть конкретный экран, скриншот). Дорого, поэтому в конце.

## Типовые причины «запушил, а в проде старое»
- **Сборка из НЕ обновлённого src на сервере.** `git pull` упал/не применился, а `build` всё равно собрал старое. ВСЕГДА проверять `git log -1` на сервере ПОСЛЕ pull, ДО build.
- **Stale git-bundle** (airgapped-деплой через bundle): старый bundle → `fatal: couldn't find remote ref master`. Пересоздать bundle, потом pull.
- **Локальные модификации на сервере блокируют pull** (правки вносили прямо на проде). Если они уже закоммичены в master — `git checkout -- <файлы>` (содержимое то же) → pull. Если нет — закоммитить/сохранить сначала.
- **Кэш браузера/CDN** — bundle сменился, но клиент держит старый: hard reload / `localStorage.clear()`.
- **Сборка статики «в образ» vs volume:** если web-контейнер baked-in static — нужен rebuild образа; копирование в host-путь не поможет (и наоборот). Проверить, что именно отдаётся.

## Антипаттерны
- ❌ «git push прошёл» = «в проде новое». Push ≠ deploy ≠ rebuild ≠ shipped.
- ❌ Закрывать задачу по факту merge без подтверждения артефакта в проде.
- ❌ Запускать `build` не проверив, что src на сервере реально обновился.
