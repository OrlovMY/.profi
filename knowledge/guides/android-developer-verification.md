# Guide: Android Developer Verification (механика регистрации пакета)

**Тип:** переносимый how-to (применимо к ЛЮБОМУ проекту, где есть Android-приложение,
распространяемое вне Play Store / через sideload / Device Owner).
**Не политика.** Решение «делать ли это обязательным для приложений конкретного продукта» —
это DoD/политика того проекта, фиксируется в проекте, а не здесь.

## Зачем

Google вводит Android Developer Verification: с сентября 2026 в ряде регионов на
**сертифицированные** устройства Android можно будет ставить только приложения от
подтверждённых разработчиков с зарегистрированными package names. Касается и sideload, и
Device Owner provisioning, не только публикации в Play Store.
Док: https://developer.android.com/developer-verification

Применимость: правило про **сертифицированные (GMS) устройства Android**. Кастомные/рутованные
несертифицированные сборки (напр. кастомные AOSP-донглы) — вероятно вне области; проверять по
официальной странице для конкретного железа/региона.

## Как зарегистрировать package (Play Console → «Проверка разработчика Android»)

1. **applicationId**: только валидные Java-идентификаторы (буквы/цифры/`_`), **без дефисов**.
   Не переименовывать существующие пакеты — смена applicationId = новое приложение (не
   обновляется поверх, ломает Device Owner binding / self-update / enroll).
2. **Названия пакетов → Зарегистрировать**: поле = точный applicationId (валидируется Google) +
   «Понятное название».
3. **Открытый ключ**: SHA-256 fingerprint signing-сертификата в colon-hex UPPERCASE
   (`AA:BB:...`). Брать из реально раздаваемого APK:
   `apksigner verify --print-certs app.apk | grep "SHA-256 digest"` → hex → разбить парами через `:`.
   Подписан несколькими ключами (легаси миграция) → добавить все актуальные отпечатки.
4. **Доказательство владения приватным ключом** (визард «Подпишите и загрузите APK»):
   - Google даёт per-package сниппет.
   - Файл `app/src/main/assets/adi-registration.properties` = **ровно сниппет одной строкой**
     (без `key=`/кавычек; формат подтверждён по
     github.com/android/security-samples/tree/main/AndroidDeveloperVerificationAPKSigningExample).
   - Собрать APK с тем же applicationId, подписать тем же приватным ключом (можно пересобрать
     существующий: добавить asset в зип → zipalign → переподписать apksigner; раздавать не нужно).
   - Загрузить → «Отправить» → Google проверяет до 48 ч, уведомление на email.

## Ограничения

- Регистрация = внешняя запись под идентичностью владельца аккаунта; делает владелец (или агент
  в его авторизованной сессии) с явным подтверждением на каждую отправку. **Новый Google-вход в
  автоматизированном CDP-браузере Google блокирует** — логинится владелец, автоматизация
  подхватывает готовую сессию.
- Тип аккаунта Personal достаточен для регистрации + верификации. Organization нужен отдельно —
  для Verified DPC / Managed Google Play EMM.
- Разные ключи подписи → разные отпечатки; зарегистрировать все ключи раздаваемого артефакта.
