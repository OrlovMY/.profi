# Guide: грабли middleware-групп в Go/Fiber

**Применимость:** любой Go-проект на Fiber (v2). Две реальные грабли, каждая роняла авторизацию/маршруты.

## Грабля 1 — пустой префикс группы протекает на чужие маршруты
```go
api.Group("", authMiddleware)   // ❌ пустой префикс
```
`Group("")` с пустым строковым префиксом регистрирует middleware **глобально на все последующие маршруты в том же router-дереве**, а не только на «свою» группу. Симптом: middleware (напр. admin-auth) налипает на маршруты, которым не предназначался (device-эндпоинты начинают требовать admin-токен → 401, «здоровые» функции мертвеют).

**Правильно:** давать группе непустой scoped-префикс:
```go
api.Group("/admin", authMiddleware)   // ✅ только /admin/*
```
Если нужно применить middleware к конкретному набору — оборачивать именно их, не «пустую» группу.

**Коварство:** может протечь не сразу, а на маршруты, добавленные ПОЗЖЕ в том же дереве. Бывает несколько таких групп — искать все.

## Грабля 2 — двойной `c.Next()` при оборачивании middleware
```go
func wrap(mw fiber.Handler) fiber.Handler {
    return func(c *fiber.Ctx) error {
        mw(c)          // mw уже зовёт c.Next() внутри
        return c.Next() // ❌ второй c.Next()
    }
}
```
Если обёртка вызывает `c.Next()`, а обёрнутый middleware тоже — следующий handler выполняется дважды или ответ теряется → **404 / странное поведение**.

**Правильно:** не дублировать `c.Next()`; решить, кто им владеет (обёртка ИЛИ внутренний mw), и звать ровно раз.

## Диагностика (обе)
- Симптом «неожиданный 401/404 на части маршрутов после добавления middleware».
- Приём: временный уникальный маркер в текст ответа 401/ошибки по веткам auth → видно, какая ветка/группа сработала.
- Проверять порядок регистрации групп и пустые префиксы (`grep 'Group("'`).

## Источник
Реальный проект: пустой префикс группы протёк auth-middleware на не предназначенные ему маршруты (массовый 401); отдельно — двойной `c.Next()` в обёртке middleware положил часть маршрутов в 404. (Конкретные пути/инциденты — в приватной памяти проекта, не здесь.)
