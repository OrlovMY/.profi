# Guide: автономная задача, переживающая /compact (и любой reset контекста)

## Условия применения
Большая многошаговая автономная задача (аудит, миграция, рефакторинг, раскатка), которая НЕ влезает в один контекст и переживёт сжатие/reset. Цель: после `/compact` точно продолжить с нужного места без потери шагов.

## Суть — 4 артефакта
1. **Живой трекер (источник истины)** — в репо/доке проекта: полная спека + фазы с чекбоксами `[ ]/[x]` + секции «CURRENT STEP», «Утверждённый план», рабочие таблицы (карта/gap/лог решений), «Журнал выполнения по шагам». Обновлять ПОСЛЕ КАЖДОГО шага (отметки + находки).
2. **Якорь в памяти** (`memory/<task>.md`, type:project) — короткий: суть задачи + правила + ссылка на живой трекер + RESUME-протокол + «CURRENT STEP сейчас = …». Память грузится в контекст каждой сессии → это и есть страховка.
3. **Указатель в `MEMORY.md`** первой строкой, заметный (🔴): «АКТИВНАЯ ЗАДАЧА X — ЧИТАТЬ ПЕРВЫМ ПОСЛЕ /compact», с путём к трекеру.
4. **(Опц.) Задача в трекере задач** для in-session видимости.

## RESUME-протокол (зашить в якорь и трекер)
1. После compact память подхватывается → читать якорь + живой трекер ПЕРВЫМ.
2. Найти «## CURRENT STEP» в трекере → продолжить ровно оттуда.
3. Сверяться со спекой-эталоном; обновлять трекер после каждого шага.

## Запуск после compact
Дать владельцу короткую фразу-триггер заранее («Старт X по трекеру»). По ней: читаю трекер → CURRENT STEP → продолжаю. Не нужно пересказывать контекст — он на диске.

## Антипаттерны
- ❌ Держать прогресс только в контексте/чате → compact сотрёт, продолжишь не с того места или повторишь.
- ❌ Трекер только в репо без указателя в памяти → после compact его не прочитаешь автоматически (репо-доки не грузятся, память — грузится).
- ❌ Обновлять трекер «в конце» → при обрыве посередине шаг потерян. Обновлять ПО ХОДУ.
- ❌ Размытый «CURRENT STEP» → после compact непонятно, откуда продолжать. Маркер должен быть однозначным (конкретная фаза/подшаг).

## Источник
Автономный многошаговый аудит, спроектированный под /compact: живой трекер в доке репо + якорь в памяти проекта + 🔴-указатель в `MEMORY.md` + однозначный CURRENT STEP + заранее согласованная фраза-триггер.
