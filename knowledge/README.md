# knowledge/ — универсальные технические знания

Кросс-проектная база знаний. Не зависит от конкретного продукта.
Пополняется, когда команда изучила что-то новое и хочет применять это везде.

## Структура

| Папка | Что хранится | Пример |
|-------|-------------|--------|
| `standards/` | Минимальные планки качества — «не ниже этого» | Минимальный UI для любого проекта, минимум безопасности APK |
| `guides/` | Критерии выбора и подходы — «как правильно» | Выбор БД под условия, стратегия сборки APK, CI pipeline setup |
| `recipes/` | Конкретные команды и последовательности — «делай так» | Gradle release build, pg_dump restore, Docker multi-stage |

## Когда писать

- Изучили новый правильный способ делать что-то → `guides/`
- Договорились о минимальном стандарте для любого проекта → `standards/`
- Нашли рабочую последовательность команд, которую будем повторять → `recipes/`

## Что НЕ сюда

- Правила работы команды → `scenarios/` или `playbooks/`
- Роли агентов → `roles/`
- Специфика конкретного проекта → `memory/` того проекта

## Формат файла

```markdown
# Тема

## Условия применения
Когда это актуально / для каких проектов.

## Суть
Что правильно и почему.

## Антипаттерны
Что делать не надо.

## Источник
Откуда знание (официальная дока, инцидент в проекте X, дата).
```
