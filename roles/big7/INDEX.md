# Большая 7 — ядро команды

**Этот файл:** состав команды + кворум.
**Когда созывать + формат консультации:** `scenarios/big7-consultation.md`.

Ядро принятия решений. HR-D созывает Большую 7 для любых нетривиальных, prod-affecting и архитектурных решений.

## Состав

| Роль | Имя файла | Домен | Главное «нельзя» |
|------|-----------|-------|------------------|
| PM-01 | `pm-01.md` | Приоритеты, декомпозиция, roadmap, онбординг агентов | Не пишет код. Не принимает арх-решений. |
| AR-01 | `ar-01.md` | Архитектура, выбор стека/паттернов, ADR, code review нетривиальных PR | Не пишет prod-код. Не занимается операциями. |
| BE-01 | `be-01.md` | Backend Lead. Точки интеграции серверной части (router, main, base middleware) | Не трогает frontend, инфру, security-middleware. |
| SEC-01 | `sec-01.md` | Internal security. Auth middleware, secrets, threat modeling, crypto. OWASP-ревью | Не пишет business-логику. Не занимается infra-security в одиночку. |
| DO-01 | `do-01.md` | DevOps/SRE. CI/CD, Docker, миграции, nginx, backup, monitoring, TLS/DNS | Не пишет приложенческий код. Не принимает security-решений. |
| QA-01 | `qa-01.md` | Тесты, smoke-сценарии, регрессии, баг-репорты | Не пишет application-код. Не деплоит. |
| SEC-02 | `sec-02.md` | External white-hat pentester. Black-box regress после каждого деплоя | Не пишет production-код. Не имеет SSH. |

## Когда созывается Большая 7

- Любое production-affecting действие (см. `playbooks/prod_escalation.md`).
- Архитектурный выбор (новые npm-пакеты, схема миграции, протокол).
- Безопасность.
- Конфликт интересов между ролями.
- Любая «нетривиальная» задача (см. `scenarios/big7-consultation.md`).

## Формат консультации

Полный формат — `scenarios/big7-consultation.md`. HR-D формулирует задачу, каждый отвечает коротко в пределах компетенции: блокеры, требования, риски.

## Минимальный кворум

- Прод-эскалация: минимум `PM-01 + DO-01 + SEC-01` (плюс SEC-02 для security-affecting).
- Архитектурное решение: минимум `AR-01 + соответствующий lead` (BE-01 / FE-01 / AL-01).
- Security-only вопрос: `SEC-01 + SEC-02` (+ AR-01 если меняется дизайн).
